3. Apply the rules of implicit promotion in arithmetic operations and the need for casting
when assigning the result.
Instruction: Indicate the final data type of the result variable and whether explicit casting is
needed to store the value.

12 int x = 5; double y = 2.0; double result = x / y;
No, the int is promoted to double before the division, since one operand is a double.

13 short a = 10; short b = 2; short result = a * b;
Yes, in arithmetic operations with types smaller than int (byte, short, char), both operands are 
promoted to int. The result is int, so casting to short is required.

14 long l = 100L; int i = 50; int result = l + i;
Yes,when combining long and int, the int is promoted to long. The result is long, and explicit casting 
is needed to store it in an int.


15 char c = 'a'; int i = 1; int result = c + i;
No, the char is promoted to int before the addition. The result is already int, so no casting is needed.


16 byte b1 = 5; byte b2 = 10; byte result = b1 + b2;
Yes, just like with short, both byte values are promoted to int in arithmetic operations, so casting 
to byte is required to store the result.
